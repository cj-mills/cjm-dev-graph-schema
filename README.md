# cjm-dev-graph-schema

The **development / decision-provenance schema** for context graphs — the
dev-domain sibling to
[`cjm-transcript-graph-schema`](https://github.com/cj-mills/cjm-transcript-graph-schema).
Where the transcript schema models *audio content*, this one models a
**project's own evolution**: the decisions, facts, evidence, threads of work,
sessions, procedures, and entities (repos, stages, capabilities) that a
software ecosystem accumulates as it is built.

It is part of the **self-hosting graph arc** — moving the ecosystem's own
decision/provenance content onto context graphs so that hand-maintained
artifacts like `MEMORY.md` become *projections* of a single-source-of-truth
graph rather than files edited by hand.

> **Born non-nbdev.** This is one of the first plain-Python (no nbdev) libraries
> in the ecosystem. Plain `.py` modules, `pytest` tests, no autogenerated
> notebooks/docs. Code is kept at fine granularity (one concept per module) so
> it maps cleanly onto graph nodes when the later *code-on-graph* slice arrives.

## Install

```bash
pip install -e .
```

Depends on
[`cjm-context-graph-layer`](https://github.com/cj-mills/cjm-context-graph-layer)
(identity / grammar / idempotent emission) and
[`cjm-context-graph-primitives`](https://github.com/cj-mills/cjm-context-graph-primitives)
(locators / `SourceRef` provenance).

## What it provides

- **`vocab`** — the reserved node-kind and edge-relation vocabulary for the
  dev domain (Decision, Fact-slot/Assertion, Evidence, Thread, Session,
  Procedure, Entity, and the coarse `Note`; edges `REFERENCES`, `ABOUT`,
  `DECIDED_IN`, `SUPPORTED_BY`, `CONTRADICTS`, …). The full vocabulary is
  reserved up front; the coarse tier is implemented first and refined into the
  fine tier as the arc progresses.
- **`identity`** — deterministic node-id helpers built on the layer's
  `derive_node_id` (a node's id derives from what makes it *the same* node
  across re-derivation, never from its correctable content).
- **`nodes`** — typed node dataclasses with `.id` / `.to_graph_node()` wire-dict
  mapping, mirroring the transcript schema's shape. `NoteNode` (the coarse
  document node) lands first.

## Status

Early — built incrementally alongside the self-hosting graph arc. The coarse
tier (`Note` + `REFERENCES`) lands first; the fine tier (Decisions, layered
Assertions, oracle-backed Fact-slots, contradiction detection) follows.
